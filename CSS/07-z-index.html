Z-INDEX

En un documento HTML existe un concepto llamado "apilamiento", es decir, los elementos dentro del HTML se pueden mostrar y sobreponer unos entre otros. Los primeros elementos en ser colocados ocuparán los primeros órdenes de apilamiento por defecto, mientras que los últimos irán sobreponiéndose si el caso se da.

Ejemplo

Los cuadrados verdosos tienen un número cada uno. Estos números representan el orden de cuando han sido implementados.

1

2

3

4

5


Sin embargo, este apilamiento no sucede de forma automática, sino, que algunas propiedades especiales deberían de estar presentes. Las cuales son:

  • Elementos con "position" igual "absolute", "relative" o "fixed".
  • Elementos flex.
  • Elementos con "opacity" menor a 1.
  • Elementos con "transform" distinto de "none".
  • Elementos con "mix-blend-mode" distinto de "normal".
  • Elementos con "filter" distinto de "none".
  • Elementos con "perspective" distinto de "none".
  • Elementos con "isolation" igual de "isolate".


Para poder verificar mejor este caso, veamos el siguiente escenario. El cuadro rojo tiene un "z-index" de 1, mientras que el cuadro verde tiene un "z-index" de 3 y, finalmente el cuadro naranja tiene un "z-index" de 2. Entonces, lo lógico es que el cuadro verde se superponga ante todos los cuadros, pero, como los elementos no se encuentran posicionados con un valor diferente a "static", el "z-index" no funciona.

1
z-index: 1
2
z-index: 3
3
z-index: 2

Pero, si aplicamos lo mismo, pero incluyendo una propiedad que permita el posicionamiento, entonces el "z-index" sí funcionará.

1
z-index: 1
2
z-index: 3
3
z-index: 2
